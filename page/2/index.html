插件钩子

通用钩子

开发时，Vite dev server创建一个插件容器按照Rollup调用创建钩子的规则请求各个钩子函数。

下面钩子会在服务器启动时调用一次：

  • options替换或操纵Rollup选项
  • buildStart开始创建

下面钩子每次有模块请求时都会被调用：

  • resolved创建自定义确认函数，常用于定位第三方依赖
  • load创建自定义加载函数，可用于返回自定义的内容
  • transform可用于转换已加载的模块内容

下面钩子会在服务器关闭时调用一次：

  • buildEnd
  • closeBundle

Vite特有钩子

  • config：修改Vite配置
  • configResolved：Vite配置确认
  • configureServer：用于配置dev server
  • transformIndexHtml：用于转换宿主页
  • handleHotUpdate：自定义HMR更新时调用

插件顺序

  • 别名处理Alias
  • 用户插件设置 enforce: 'pre'
  • Vite核心插件
  • 用户插件未设置 enforce
  • Vite构建插件
  • 用户插件设置 enforce: 'post'
  • Vite构建后置插件（minify， manifest， reporting）

Promise

异步编程解决方案。

三种状态

pending 待定

fulfilled 兑现

rejected 拒绝

使用

接收两个参数：resolve、reject 。resolve成功的回调，会通过onResolved函数进行处理。reject拒绝的回调，使用onRejected函数处理。

Promise.resolve()

传入的参数不是一个期约对象

使用Promise.resolve()，如果传入的参数不是一个期约实例那么总是返回解决状态的期约。例如：

上面都是传递一个参数，如果传递多个参数，Promise.resolved()也只接收一个参数，忽略其他的参数

传入的参数是一个期约

Promise.resolved()相当于一个幂等函数

Promise.reject()

Promise.reject()会实例化一个拒绝的期约，并且抛出一个异步错误。该错误只能通过拒绝处理程序（onRejected、.catch）捕获，不能通过try/catch捕获。

传入参数不是期约对象

传入参数是一个期约对象

Promise.reject()参数如果是一个期约对象，则会把这个期约对象作为拒绝的理由，而不是幂等。

期约的实例方法

  • Promise.prototype.then()
  • Promise.prototype.catch()
  • Promise.prototype.finally()

Promise.prototype.then()

then方法接收两个回调函数，分别是对解决状态的回调（主动调用resolve()）onResolved(),对拒绝状态的回调（主动调用reject()）onRejected。

如果没有对应的回调可以传入null

then方法也返回一个Promise实例,通过Promise.resolve()包装生成新期约

MVC MVVM
