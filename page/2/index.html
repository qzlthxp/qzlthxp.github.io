下面钩子会在服务器启动时调用一次：

  • options替换或操纵Rollup选项
  • buildStart开始创建

下面钩子每次有模块请求时都会被调用：

  • resolved创建自定义确认函数，常用于定位第三方依赖
  • load创建自定义加载函数，可用于返回自定义的内容
  • transform可用于转换已加载的模块内容

下面钩子会在服务器关闭时调用一次：

  • buildEnd
  • closeBundle

Vite特有钩子

  • config：修改Vite配置
  • configResolved：Vite配置确认
  • configureServer：用于配置dev server
  • transformIndexHtml：用于转换宿主页
  • handleHotUpdate：自定义HMR更新时调用

web模板

基本结构

  1. html标签是由<>包围的关键词
  2. html标签通常成对出现，分为标签开头和标签结尾。（

    ）
  3. 有部分标签是没有结束标签的，为单标签，单标签必须使用/结尾。（ ）
  4. 页面所有的内容，都在html标签中
  5. html标签分为三部分：标签名称，标签内容，标签属性
  6. html标签具有语义化，可通过标签名能够判断出该标签的内容，语义化的作用是网页结构层次更清晰，更容易被搜索引擎收录，更容易让屏幕阅读器读出网页内容
  7. 标签的内容是在一对标签内部的内容
  8. 标签的内容可以是其他标签

标签属性

  1. class属性：用于定义元素的类名
  2. id属性：用于指定元素的唯一id，该属性的值在整个html文档中具有唯一性
  3. style属性：用于指定元素的行内样式，使用该属性将会覆盖任何全局的样式设定
  4. title属性：用于指定鼠标悬停在标签内容时，显示的信息
  5. accesskey属性：用于指定激活元素的快捷键
  6. tabindex属性：用于指定元素在 tab 键下的次序
  7. dir属性：用于指定元素中内容的文本方向，属性只有 ltr 或 rtl 两种。
  8. lang属性：用于指定元素内容的语言

事件属性

  1. window窗口事件，onload，在网页加载结束之后触发，onunload，在用户从网页离开时发生（点击跳转，页面重载，关闭浏览器窗口等）

  2. from表单事件，onblur，当元素失去焦点时触发，onchange，在元素的值被改变时触发，onfocus，当元素获取焦点时触发，onreset，当表单重置按钮被点击时触发，onselect，在元素中文本被选中后触发，onsubmit，在提交表单时触发

  3. keyboard键盘事件，onkeydown，在用户按下按键时触发，onkeypress，在用户按下按键后，按着按键时触发，该属性不会对所有按键生效，不生效的有，alt，ctrl，shift，esc

  4. mouse鼠标事件

    事件名称 触发条件 影响
    onclick 当在元素上发生鼠标点击时触发 点击子元素，事件会冒泡到父元素
    ondblclick 当在元素上发生鼠标双击时触发 双击子元素，事件会冒泡到父元素
    onmousedown 当在元素上按下鼠标时触发 在子元素上按下鼠标，事件冒泡到父元素
    onmouseup 当在元素上按下鼠标松开后触发 在子元素上松开鼠标，事件会冒泡到父元素
    onmouseenter 当鼠标移入元素时触发 正常情况：移入谁触发谁的事件。特殊情况：鼠标一开始就在元素上，那么移动鼠标先触发父元素事件，再触发当前元素事件。
    onmouseover 当鼠标在元素上时触发 在子元素上会冒泡到父元素
    onmousemove 当鼠标在元素上移动时触发 在子元素上移动会冒泡到父元素
    onmouseleave 当鼠标移出元素时触发 移除谁触发谁的事件
    onmouseout 当鼠标移出元素时触发 首先鼠标移入父元素，然后继续移入子元素这是会触发一次父元素事件，移出子元素先触发子元素事件，然后触发一次父元素事件，最后移出父元素再触发一次父元素事件

文本标签

  1. 段落标签<p></p>，描述一段文字
  2. 标题标签<hx></hx>，标题标签用来描述一个标题，标题标签总共有六个级别h1~h6，h1一般每个页面只出现一次
  3. 强调语句标签<em></em>，用于强调某些文字的重要性
  4. 更加强调标签<strong></strong>，用于强调文本，但它强调的程度更强一些
  5. 无语义标签<span></span>
  6. 短文本引用标签<q></q>,简短文字的引用
  7. 长文本引用标签<blockquote><blockquote>，定义长的文本引用
  8. 换行标签<br>

多媒体标签

网页结构

  1. <!DOCTYPE html>定义文档类型，告知浏览器用哪一种标准解释HTML
  2. <html><html>告知浏览器其自身是一个HTML文档
  3. <body></body>标签之间的内容是网页的主要内容
  4. <head></head>用于定义文档的头部，它是所有头部元素的容器
  5. <title></title>定义文档的标题
  6. <link>标签将css样式文件链接到HTML文件内
  7. <meta>定义文档的元数据

CSS代码语法

  1. css全称为层叠样式表，它主要用于定义HTML内容在浏览器内的显示样式，如文字大小、颜色、字体加粗等
  2. css代码通常放在<style></style>
  3. css样式由选择符和声明组成，而声明又由属性和值组成
  4. 选择符{属性：值}
  5. 选择符：又称选择器，指明网页中要应用样式规则的元素

CSS放置位置

  1. 行内样式，不建议使用
  2. 内联样式表
  3. 外联样式表(需要引入)

CSS的继承

  1. css的某些样式具有继承性的，继承是一种规则，它允许样式不仅应用于某个特定html标签元素，而且应用于其后代
  2. 不可继承的样式：dispaly、margin、border、padding、background、height、min-height、max-height、width、min-width、max-width、overflow、position、left、right、top、bottom、z-index、float、clear
  3. 可以继承的样式：color、font-size、font-family、font-weight、text-align、white-space、line-height、visibility

选择器的种类

  1. 标签选择器：通过标签的名字，修改css样式（p、h1、main、header、span）
  2. 通配符选择器：选择页面中所有的元素（*）
  3. 属性选择器：a[href][title]{color: red;}将同时有href和title属性的HTML超链接的文本设置为红色
  4. 后代选择器：选择某个父元素下面的所有子元素（main>section）
  5. id选择器：通过id查找页面中唯一的标签（#app）
  6. class选择器：通过特定的class来查找页面中对应的标签（.app）
  7. 伪类选择器：（:hover、:active、:nth-child()、:nth-of-type()、:last-child）
  8. 伪元素选择器：（::before、::after、::first-letter、::first-line）

选择器的优先级

  1. 当有不同的选择器同时对一个对象进行样式指定时，并且两个选择器有相同的属性被赋予不同的值时
  2. 计算权重

为什么打印的结果是每次count加1后的结果

首先先写一个常规累加函数

上述代码讲解：

  1. 定义了一个全局变量count，这个变量存在于全局执行上下文，在页面没有关闭前，这个变量一直存在在内存中，可以对这个变量进行赋值、自增等操作，而且每次打印这个count变量都是上次更改后的结果。
  2. 代码执行到inc()调用inc这个函数。会将这个inc()函数的执行上下文压入上下文栈中。创建上下文有两个阶段：创建阶段和执行阶段。创建阶段创建变量对象、创建作用域链、创建this这三个操作。执行阶段就是执行函数内部代码。
  3. inc函数通过作用域链访问到全局变量count，并且每次调用都会使count加1，执行完inc函数，inc被弹出上下文栈，浏览器也进行垃圾回收。

代码问题：

  1. 使用全局变量进行计算，很容易造成变量被修改的问题，不安全。

这次把变量放入函数内部，保证不会被随意修改

问题出现了，调用inc函数并没有达到我们所需要的在上次的基础上进行加1，而是每次都是对0进行加1。

讲解：

  1. 同样调用inc函数，创建上下文并压入上下文栈中。
  2. count在函数内部声明，每次执行完函数退出上下文栈，函数内部变量就会被销毁，因此无法做到累加。

因此需要让这个内部count一直存在内存中，只需要使count一直被引用这样count一直存在内存中不会被回收。第一种方法，就是全局作用域声明变量我们已经说过了，第二种在全局上下文声明一个函数这个函数引用count并且实现自增效果。

代码问题：

​ 使用这种形式，解决了在全局作用域直接声明变量造成的安全性问题，但是由于一直引用这个count变量，只有关闭页面才会释放内存。这也造成了内存泄露（内存无法回收。造成内存泄露的情况：引用没有声明的变量、setInterval、闭包）。

基础面试题

HTML面试题

  • doctype的作用是什么？

    声明文档类型，告诉浏览器用什么模式解析文档。标准模式和怪异模式

  • 这两种模式的区别是什么?

    标准模式页面按照HTML与CSS定义渲染

    怪异模式会模拟更旧的浏览器行为

  • 什么是data-属性？

    用户自定义属性

  • 你对HTML语义化的理解？

    根据内容选择合适的标签，有利于搜索引擎收录、提高阅读性和维护性

  • HTML5与HTML4的不同之处

    声明方式、不在基于sgml、新的语义标签、input新的type

  • 有哪些常用的meta标签？

    charset设置编码方式、viewport兼容移动端、author声明作者、keywords关键词利于搜索、http-equiv相当于http的文件头作用，可以设置http的缓存过期日期

  • src和href的区别

    src引入外部文件、href引入网络资源

  • 知道img的srcset的作用是什么？

    可以设计响应式图片，我们可以使用两个新的属性srcset和sizes来提供更多额外的资源图像和提示，帮助浏览器选择一个正确资源。

  • script标签中defer和async的区别？

    都是异步加载，defer等待文档解析完执行，并且按照顺序执行。async加载完就会执行没有顺序。

  • 有几种前端存储方式？

    cookie、sessionStorage、localStorage、indexdb

  • 这些方式的区别是什么?

    sessionStorage:会话结束就会自动移除、localstorage：永久存储需要手动清除、cookie可以设置过期时间但是浏览器可能会禁用cookie

CSS面试题

  • css选择器的优先级是怎样的？

    计算权重通配符是0，元素是1，class是10，id是100，行内为1000，！import最高

  • link和@import的区别？

    link属于html的一个标签兼容性好，不仅可以引入样式还能引入图片和其他资源，随文档一起加载。@import只能用于引入样式文件，并且是页面加载完成后引入

  • 有哪些方式可以隐藏页面元素?

    opacity: 0、visibility: hidden、这两个设置隐藏还是会占用物理空间。display：none不会占用物理空间

  • em\px\rem的区别？

    px屏幕像素设置多少就显示多少。em根据父元素的font-size来计算。rem根据根元素（:root和html）上设置的font-size来计算

  • 块级元素水平居中的方法？

    margin：0 auto；(需要设置width)

    position: absolute; left: 50%; transform: translateX(-50%);

    父元素开启flex display: flex; justify-content: center;

  • CSS有几种定位方式?

    static静态设置静态后top right bottom left不起作用

    relative 相对于元素原来的位置

    absolute相对于父元素，父元素也需要设置定位

    fixed相对于窗口

    sticky到达设定的位置变为fixed

  • 如何理解z-index？

    层级。值越大显示在上面

  • 如何理解层叠上下文？

    一个元素后设置的属性会覆盖前面设置的属性

  • 清除浮动有哪些方法？

    clear、overflow除了visible、position：absolute

  • 你对css-sprites的理解？

    减少网络请求，获得图片通过background-position去调整图片位置

  • 你对媒体查询的理解？

    通过@media设置条件，窗口达到特定条件就使用@media中的样式

  • 你对盒模型的理解？

    content-box 设置的宽高为内容宽高，改变内边距和边框宽度会改变整个盒子的大小，容易溢出

    border-box 设置的宽高就是盒子宽高，改变内边距和边框宽度不会改变盒子大小

  • 谈谈对BFC（Block Formatting Context）的理解？

    块级格式化上下文，创建一个独立的渲染区域。用于处理高度坍塌、相邻元素外边距叠加问题

  • 为什么有时候人们用translate来改变位置而不是定位？

    改变transform不会触发浏览器回流或重绘，只会触发复合。

  • 伪类和伪元素的区别是什么？

    伪类前面是单冒号通常用来描述元素的状态，伪元素是双冒号是一个元素插入到父元素中，只有闭合标签才有伪元素

  • 你对flex的理解？

    flex弹性盒模型，通过主轴justify-content和交叉轴align-item设置容器内元素的排列。

  • 关于CSS的动画与过度问题?

    过度transition设置元素从一个状态到另一个状态以怎样的方式进行过度，不能设置过度过程中元素样式。animation关键帧动画可以设置关键帧下元素状态

js面试题

  • 谈谈你对原型链的理解？

    使用原型实现继承，实例化对象的原型指向构造函数的原型，构造函数的原型有指向它继承的构造函数的原型，形成原型链。实例化对象可以访问原型链上的属性和方法。

  • 如何判断是否是数组？

    arr instanceof Array 检测Array的原型是否在arr的原型链上

    Array.isArray()

    arr.construct === Array

  • ES6模块与CommonJS模块有什么区别？

    CommonJS 模块输出的是一个值的拷贝，ES6 模块输出的是值的引用。
    CommonJS 模块是运行时加载，ES6 模块是编译时输出接口。

  • 聊一聊如何在JS中实现不可变对象？

    Object.defineProperty

  • js的参数是按照什么方式进行传递的？

    按基本类型传递、按复杂类型传递、按共享传递

  • null和undefined的区别是什么？

    null表示没有值原型链的最顶端，undefined表示缺少值，访问一个对象不存在的属性返回undefined

  • 解释下变量提升？

    解析代码使用var声明的变量在代码执行前会在变量所在作用域的最顶端进行声明

  • 一段js代码如何执行的？

    首先预加载解析代码进行变量提升和函数提升，运行时按照同步代码、微任务、宏任务的顺序执行

  • 谈一谈你对this 的理解？

    this指向调用这个方法的对象，箭头函数的this指向外层上下文的this

  • 理解闭包吗？

    闭包函数外部访问函数内部变量。调用函数创建上下文压入调用栈执行内部代码返回一个匿名函数，由于这个匿名函数被全局变量引用所以不会被回收。造成内存泄漏

  • 造成内存泄漏的几种情况？

    对一个没有声明的变量赋值，这个变量会加入全局作用域。

    setInterval如果不清除定时器也会造成内存泄漏

    闭包

  • 讲讲js的垃圾回收是怎么做的？

    引用计数、标记清理

  • js的基本类型和复杂类型是存储在哪里的？

    js分为栈内存和堆内存。基本类型存放在栈内存，复杂类型存储在堆内存，栈内存存放这个对象的内存地址

  • async/await是什么？

    async声明一个异步函数，await处理promise相当于.then使用try catch捕获错误。await只能在async声明的函数中使用

  • async/await相比于Promise的优势？

    简化了链式操作

浏览器与新技术面试题

  • 浏览器如何渲染UI的？

    解析html文件形成dom树

    解析css形成样式规则

    dom树于样式规则结合形成render树

    为每个节点分配一个应出现在屏幕上的确切坐标

    绘制

  • 浏览器如何解析css选择器？

    从右往左解析css选择器

  • DOM Tree是如何构建的？

    1. 转码： 浏览器将接收到的二进制数据按照指定编码格式转化为HTML字符串
    2. 生成tokens： 浏览器将HTML字符串解析成tokens
    3. 构建nodes： 对Node添加特定的属性，通过指针确定Node的父、子、兄弟关系和所属treeScope
    4. 生成DOM Tree： 通过node包含的指针确定的关系构建dom tree

    浏览器同源策略，协议、IP、端口不同不允许通信。配置反向代理，后端CORS跨域资源共享

Vue面试题

  • 你对MVVM的理解？

    model模型、view视图、viewModel视图模型。

    模型和视图没有直接联系。数据改变会直接反应到视图，视图操作也会影响数据

  • Vue生命周期的理解？

    vue实例从创建、初始化、挂载、更新、消亡的一个过程。不同过程也提供相应的hook函数方便用户操作。

    beforeCreate： 实例初始化之后，此时this指向vue实例

    created： 实例创建完成可以操作data里的数据和发送网络请求

    beforeMount： 实例挂载之前调用不能获取dom

    mounted：实例挂载完成可以通过$refs获取dom

    beforeUpdate： 更新之前调用此时data中数据发生变化但是视图还没有更新

    updated：更新后调用可以通过$nextTick获取更新后的dom，这个阶段不能修改数据会造成死循环

    beforeDestory：实例销毁之前调用，此时实例依旧可用，可以做一些removeEventListener操作

    destoryed： 实例被销毁

  • Vue组件如何通信？

    1. 父子组件通信 子元素上动态绑定属性，子元素内部通过props接收参数，子元素$emit()发送事件父元素中子元素v-on绑定事件
    2. 非父子组件通信： $attr、$listener、vuex、bus总线

  • computed和watch的区别？

    计算属性是多个值影响一个值，通常做一些过滤操作。watch是一个值影响其他值，监听数据变化进行一些操作

  • vue如何实现响应式？

    数据拦截结合发布者订阅者模式Object.definePorperty setter设置值getter获取值

  • Proxy与Object.definePorperty 的优劣对比？

    Proxy优势：可以直接监听对象而非属性、Proxy可以直接监听数组的变化、Proxy多达13种拦截方法、Proxy返回一个新对象，而Object.definePorperty只能遍历对象属性直接修改
